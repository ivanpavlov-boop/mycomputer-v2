# APCOM Preview-only Feed Profile Design

## Status

Phase 9C.6.5C.3B.1 completed, merged, and synced its read-only operational
preview. This v1 profile remains an immutable historical contract.

The profile key is `apcom-preview-feed-profile-v1`. It is static,
non-persisted, read-only, and non-executable. It references
`apcom-human-decisions-v1` and the existing
`apcom-observed-stock-v1` review semantics.

## Command

`suppliers:design-preview-feed-profile` is a local-source, review-only command.
It calls the existing local source-to-staging reconciler directly; it does not
shell out to another Artisan command. It accepts only an explicitly supplied,
SHA-256-pinned local XML source plus the existing expected supplier, schedule,
import, staging, linked, unlinked, and timestamp baseline arguments.

The command supports table or JSON output, bounded sample limits, and the
standard `--no-interaction` and `--no-ansi` console options. It intentionally
does not expose `--apply`, `--persist`, import, sync, CRUD, network, schedule,
queue, image, or destructive source-absence controls.

Before it reports aggregates, the shared reconciler verifies the supplier
baseline, schedule freeze, no active import, local-file safety, source hash,
protected-table fingerprints, and safe Catalog Sync flags. CREATE remains
enabled while UPDATE, Sync All, and automatic sync remain disabled.

## Field Design

| Meaning | Source observation | Role in C3B |
| --- | --- | --- |
| Supplier SKU | `xml.product.partno` | Exact source-to-staging identity only. |
| EAN/GTIN | `xml.product.ean` | Diagnostic only. |
| Name, brand, category | `name`, `manufacturer`, `category` | Presence only; no overwrite. |
| Stock | `xml.product.stock` | Unresolved numeric observation. |
| Quantity and availability | No approved mapping | Unresolved. |
| EOL | `xml.product.eol` | Review only. |
| DAC and FD | `dac_price`, `fd_price` | Price candidates only. |
| Selected price, currency, VAT, Green Tax, MPN | No approved mapping | Unresolved. |
| Images, description, CN code, group | Existing source paths where present | Presence only; no import or mapping. |

The design does not emit raw source values. Bounded diagnostic samples remain
domain-separated SHA-256 hashes generated by the existing reconciler.

## Aggregate Candidate Classes

The report can count, but cannot execute, these classifications:

- exact match, source-only, staging-only, matched-linked, and matched-unlinked;
- linked and unlinked staging-only groups;
- EOL review, zero-price review, blank-EAN review, and EAN-conflict review; and
- unresolved-stock review.

`would_create`, `would_update`, `would_delete`, `would_link`, and
`would_unlink` are labels for aggregate preview classification only. They do
not authorize actions and all protected `records_changed` counters remain zero.

## Safety Guarantees

The success verdict is `preview_feed_profile_requires_human_decisions`: it
means the local proof is safe and complete enough for review, not that the
pending business decisions are approved.

The report always states that human review is required and that no profile was
persisted, no executable import configuration was created, no import ran, no
Catalog Sync ran, no links changed, no schedule changed, and no image was
imported. It does not write suppliers, `supplier_products`, products,
categories, mappings, families, attributes, import history, or Catalog Sync
tables.

The report says no profile was persisted, no import ran, and no Catalog Sync ran.

No real APCOM XML, production database, operational report, VPS, external feed,
credential, or remote endpoint is read by the C3B implementation or its tests.
Synthetic fixtures are used only for local test coverage.

## Non-approval Statement

C3B does not approve a feed profile, persistence, import, schedule re-enable,
stock/quantity/availability interpretation, price selection, currency, VAT,
Green Tax, MPN mapping, linking, unlinking, deletion, content overwrite, image
import, Sync All, automatic sync, or UPDATE sync.
